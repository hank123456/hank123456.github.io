hexo搭建过程

操作系统是win7。

先安装nodejs

  • step1：安装。现在的版本简单了。直接官网（国内可以去这个http://nodejs.cn）下载安装，一路next即可。

  • step2：熟悉npm。看这个npm入门文档（https://segmentfault.com/a/1190000005799797）。

  • step3：使用淘宝的镜像*（https://npm.taobao.org/），国内镜像速度快。

遇到的问题：
npm安装任何东西都失败。
解决办法：cmd使用管理员权限打开。

然后安装hexo

  • 命令：cnpm install -g hexo 注意：-g是指全局安装hexo。
  • 安装完成后，在你准备文件夹下（如D:\Hexo），执行以下指令:hexo init。(在d:\Hexo内点击鼠标右键，选择Git Bash)，Hexo 即会自动在目标文件夹建立网站所需要的所有文件。
  • 进入上面创建的文件夹：hexo generate 生成。
  • hexo server 可以打开本地服务。默认端口4000.如果端口被占可以换个端口。命令举例:hexo server -p 4001。
  • 这样打开 http://localhost:4001/ 就能看到你本地博客搭建成功。

创建GitHub仓库

  • 创建的时候注意Repository的名字。比如我的Github账号是hank123456，那么我应该创建的Repository的名字是：hank123456.github.io。
  • 然后配置，具体参考 git 使用（ssh key创建和github使用）

修改 Hexo 中的_config.yml 文件

  • 找到其中的 deploy 标签，改成下图所示形式，并保存。注意：冒号后面要加上一个空格，否则会报错。
    deploy: type: git
    repo: git@github.com:你的名字/你的名字.github.io.git
  • 然后是这些命令:
    hexo clean
    hexo generate
    hexo deploy
  • 如果报错则先： npm install hexo-deployer-git –save 再执行前面命令

到这一步，如果没有再报错，就能看到git将你public文件夹下的内容上传，个人博客就已经部署到 GitHub 上了。你可以到你的GitHub仓库查看是否已经更新。此时,通过yourUserName.github.io（即你那个仓库的名称，形如：”你的 GitHub 用户名”.github.io）,就可以看到你的个人博客了。
