hexo搭建过程

操作系统是win7。

先安装nodejs

  • step1：安装。现在的版本简单了。直接官网（国内可以去这个http://nodejs.cn）下载安装，一路next即可。

  • step2：熟悉npm。看这个npm入门文档（https://segmentfault.com/a/1190000005799797）。

  • step3：使用淘宝的镜像*（https://npm.taobao.org/），国内镜像速度快。

遇到的问题：
npm安装任何东西都失败。
解决办法：cmd使用管理员权限打开。

然后安装hexo

  • 命令：cnpm install -g hexo 注意：-g是指全局安装hexo。
  • 安装完成后，在你准备文件夹下（如D:\Hexo），执行以下指令:hexo init。(在d:\Hexo内点击鼠标右键，选择Git Bash)，Hexo 即会自动在目标文件夹建立网站所需要的所有文件。
  • 进入上面创建的文件夹：hexo generate 生成。
  • hexo server 可以打开本地服务。默认端口4000.如果端口被占可以换个端口。命令举例:hexo server -p 4001。
  • 这样打开 http://localhost:4001/ 就能看到你本地博客搭建成功。

创建GitHub仓库

  • 创建的时候注意Repository的名字。比如我的Github账号是hank123456，那么我应该创建的Repository的名字是：hank123456.github.io。
  • 然后配置，具体参考 git 使用（ssh key创建和github使用）

修改 Hexo 中的_config.yml 文件

  • 找到其中的 deploy 标签，改成下图所示形式，并保存。注意：冒号后面要加上一个空格，否则会报错。
    deploy: type: git
    repo: git@github.com:你的名字/你的名字.github.io.git
  • 然后是这些命令:
    hexo clean
    hexo generate
    hexo deploy
  • 如果报错则先： npm install hexo-deployer-git –save 再执行前面命令

到这一步，如果没有再报错，就能看到git将你public文件夹下的内容上传，个人博客就已经部署到 GitHub 上了。你可以到你的GitHub仓库查看是否已经更新。此时,通过yourUserName.github.io（即你那个仓库的名称，形如：”你的 GitHub 用户名”.github.io）,就可以看到你的个人博客了。

github page搭建的博客，怎么绑定自己的域名

怎么用hexo建站，网上有大把的教程，我也是按照教程弄出来，大家可以自行搜索。

终于建站好了发现域名好丑（太长），于是决定买个域名绑定。最终选择的是万网的，比较了几个国内外的域名商，比如godaddy，发现国内的价格更低，操作更方便。

绑定方法如下：

  • 1、在source文件夹（不是主题的source）中新建一个CNAME文件（无后缀名），然后用文本编辑器打开，在首行添加你的网站域名，如www.hanksu.com，特别注意前面没有http:// ，www有没有好像都行，我是有的，这个可以自行测试，然后使用hexo g && hexo d上传部署。然后你会发现在你的github repository里面根目录下面会多一个CNAME文件。

  • 2、进入万网后台，域名管理下面，点击对应域名的解析按钮。
    a）先添加一个CNAME，主机记录写@，后面记录值写上你的
    xxxx.github.io
    b）再添加一个CNAME，主机记录写www，后面记录值也是xxxx.github.io
    这样别人用www和不用www都能访问你的网站（其实www的方式，会先解析成http://xxxx.github.io ，然后根据CNAME再变成http://xxx.com ， 即中间是经过一次转换的）。

  • 3、等几分钟左，就可以用你的域名访问了。

PS：以上，我们用的是CNAME，也有人使用A记录，后面的记录值是写github page里面的ip地址，不过有时候IP地址会变，导致解析不正确，所以还是推荐用CNAME别名记录要好些，不建议用IP。

PPS：用CNAME的时候，如果遇到你的域名也有邮箱的域名记录，可能CNAME会和MX记录冲突，这个暂时无解，只能二选一了。如果一定要用邮件，那么还是用A记录绑定吧。

常用域名记录解释：A记录、MX记录、CNAME记录、TXT记录、AAAA记录、NS记录

刚建好blog，需要绑定域名，没想到遇到CNAME和MX冲突的问题，于是好好了解了下这几个域名记录的相关解释。以下内容详情来自网络，原作者未知，hank整理。

A记录

A记录是用来创建到IP地址的记录。

A记录设置技巧

1、如果想创建不带www的记录，即hanksu.com，在主机记录中填写@或者留空白。

2、创建多个域名到同一个IP，比如给博客建了二级域名，可以使用*.blog.hanksu.com来指向一个IP，这样的话，不管是访问a.blog.hanksu.com还是b.blog.hanksu.com都能到同一个IP。

3、如果你给同一个二级域名设置了多个A记录，比如你建了两个blog的A记录，其中一个指向了111.111.111.111，另一个指向了111.111.111.222，那么在查询的时候，每次返回的数据包含了两个IP地址，但是在返回的过程中数据排列的顺序每次都不相同。由于大部分的客户端只选择第一条记录所以通过这种方式可以实现一定程度的负载均衡。

在命令行下可以通过nslookup -qt=a www.hanksu.com来查看A记录。

MX记录

在命令行下可以通过 nslookup -qt=mx hanksu.com 来查看MX记录。我用的是万网，在网页上直接设置就行了。

mx 记录的权重对 Mail 服务是很重要的，当发送邮件时，Mail 服务器先对域名进行解析，查找 mx 记录。先找权重数最小的服务器（比如说是 10），如果能连通，那么就将服务器发送过去；如果无法连通 mx 记录为 10 的服务器，那么才将邮件发送到权重为 20 的 mail 服务器上。

这里有一个重要的概念，权重 20 的服务器在配置上只是暂时缓存 mail ，当权重 20 的服务器能连通权重为 10 的服务器时，仍会将邮件发送的权重为 10 的 Mail 服务器上。当然，这个机制需要在 Mail 服务器上配置。
